# Build Card Status Guide

Build cards need status labels because Albion balance changes and workbook packages evolve.

## Status labels

| Status | Meaning | Public use |
|---|---|---|
| Draft | Early structure or text-only card | Do not treat as final |
| Workbook-backed | Matches a current workbook package | Safe for package users, still reviewable |
| Reviewed | Checked by trusted role/build reviewers | Safer for broader sharing |
| Stable | Public-ready until patch/build changes | Good public reference |
| Outdated | Preserved for history only | Do not use |

## Patch sensitivity

All item/build pages should be treated as patch-sensitive. The page may teach good role logic, but item names, cooldowns, ability choices, and exact values can change.

## Rule

A build card should answer:

- what role is this?
- what package supports it?
- what does it do in the fight?
- what timing mistake ruins it?
- what pages should the player read next?
