# Sources and Credit

The Albion ZvZ Manual is a teaching project first.

Its purpose is to help Albion players learn ZvZ through practical role guides, fight concepts, examples, training methods, and review tools. Sources and credits explain where the teaching foundation comes from and help keep the project honest.

## Core credit

Core educational credit for the AO University doctrine belongs to **Khelron** and **AO University**.

This project does not claim ownership of Khelron's teachings. It organizes and applies those teachings, along with project synthesis and Formation Ledger application, into a public manual for the wider Albion community.

## How sources are used

Sources should support the guide. They should not overwhelm the reader path.

Normal teaching pages do not need to credit every example individually. The guide should only call out source status when it helps the reader understand confidence, privacy, or patch sensitivity.

## Source layers

| Source layer | Use |
|---|---|
| AO University public YouTube material | main doctrine backbone |
| Private/unlisted AO University guild VOD reviews | anonymized practical patterns |
| Albion Wiki / official references | mechanics verification |
| Patch notes / test-server / balance-change references | current item, ability, or mechanics checks when exact values matter |
| Community references | terminology, common gaps, review context |
| EVE-style documentation references | structure and information architecture |
| Formation Ledger application | comp-specific operating tools |

## What is public doctrine vs project synthesis?

**Public doctrine** is the broader ZvZ teaching extracted from AO University/Khelron material.

**Project synthesis** is our organized interpretation: learning paths, page structure, Formation Ledger packages, diagrams, templates, and review workflows.

Both should be useful, but they should not be confused.

## Weapon examples are teaching examples

A weapon example usually exists to teach a function:

- area denial
- stop pressure
- burst timing
- sustained pressure
- healing coverage
- utility layering
- mobility support
- purge, pierce, anti-heal, or disruption windows

Unless a page explicitly says otherwise, examples are not universal current-meta recommendations.

## VOD privacy rule

Private/unlisted guild review material should be used only as anonymous teaching patterns unless explicit permission is given.

Do not publish:

- private player names
- private guild criticism
- unlisted VOD identifiers
- direct critique that identifies a person
- screenshots with names

## Mechanics verification rule

Mechanics can change. Exact values should be verified against current Albion sources before stable publication when they matter to the claim.

Use stable wording where possible:

> AoE escalation makes clumping more dangerous.

Avoid unsupported hard numbers unless recently checked.
