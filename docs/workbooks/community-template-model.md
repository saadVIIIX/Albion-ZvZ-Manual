# Community Template Model

The workbook should serve the public Albion community without becoming a guild-only operating system.

## Core idea

The public workbook should be a **copyable template**.

A guild, alliance, faction group, or open community can copy it and adapt it to their own content.

## What the public template should do

- teach signup structure
- show party/role assignment logic
- check healer/support/tank coverage
- provide build-library structure
- connect to website role pages
- provide training and VOD review tabs
- include public-copy instructions

## What it should not do

- require membership in a specific guild
- include private player data
- assume one alliance command structure
- include private authentication/services
- expose officer-only notes
- pretend one comp package fits all Albion content

## Private/live copies

Groups can make their own private live copies.

Those copies may include:

- player names
- officer notes
- attendance
- private CTA notes
- internal build variants
- guild-specific expectations

But those belong to the group, not the public template.

## Brave/EVE reference translation

EVE alliance sites can inspire:

- onboarding paths
- doctrine package organization
- operational checklists
- review workflows
- public/private separation

They should not be copied directly as:

- membership auth
- guild services
- internal alliance rules
- private fleet permission systems

This project remains a public Albion learning project first.
