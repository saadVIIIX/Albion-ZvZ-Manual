# Workbook Package Depth Model

The workbook is a reusable operating template, not a promise that every Albion comp can immediately become a full doctrine package.

## Package depth tiers

| Tier | What it contains | Source requirement |
|---|---|---|
| Full Doctrine Package | builds, scaling, role logic, fight style, review notes, training notes, common mistakes, counters | strong comp-specific doctrine and/or repeated VOD review data |
| Lite Comp Package | builds, role purpose, basic scaling, basic warnings | some source confidence, but not full doctrine |
| Build Sheet Only | builds, signups, party structure | low doctrine confidence or basic organizational use |
| Research / Placeholder | ideas only | not ready for public use |

## Current package status

| Package | Current tier | Why |
|---|---|---|
| Great Fire Control Formation | Full Doctrine Package | supported by AOU teachings, comp guide material, private VOD patterns, scaling logic, and role notes |
| Future comp packages | Not active by default | require source-backed data before being promoted |

## Why this model matters

The Great Fire Control Formation package has unusual source depth. Most groups will not have that much comp-specific review data.

That is okay.

The workbook can still show the community what a serious comp package **could** look like when enough data exists.

## Community contribution model

A group that wants a new package structured in this system should bring:

- comp purpose
- party structure
- builds
- scaling order
- role responsibilities
- common mistakes
- caller logic
- fight-style notes
- VOD lessons or examples
- known weaknesses / counters
- source confidence

Without that, the package should stay Lite, Build Sheet Only, or Research.

## Public promise

The project should not promise a full library of every Albion comp.

It should promise:

> a public ZvZ manual, one flagship full-doctrine workbook package, and a reusable structure for future packages if the community brings enough data.
