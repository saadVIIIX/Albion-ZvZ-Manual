# Website + Workbook Model

The website and workbook should not compete with each other.

## Website job

The website teaches:

- mechanics
- role doctrine
- fight concepts
- playstyle literacy
- training methods
- sources and credit
- comp workbook navigation

## Workbook job

The workbook runs content:

- signups
- party assignment
- build library
- coverage checks
- role quick notes
- training checklist
- VOD review checklist

## Archive job

The archive preserves:

- transcript/VOD extraction
- source-heavy notes
- privacy rules
- project decisions
- handoff data for future chats

## Future comp packages

As the project grows, the website can become the hub for multiple workbook packages:

- Great Fire Control Formation
- Ranged Pressure
- Kite / Control
- Melee Clap
- Melee DoT / Pressure Brawl
- Bomb Squad
- Castle / Territory

A comp workbook should only be created when there is enough reliable doctrine, build data, and practical need.
