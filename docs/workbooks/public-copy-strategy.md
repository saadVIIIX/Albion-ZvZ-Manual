# Workbook Public-Copy Strategy

The live workbook and the public workbook should not be the same file by default.

A live workbook can contain officer notes, internal experiments, player names, draft tabs, and unfinished admin scaffolding. A public copy should be cleaned before sharing.

## Public-copy goal

Create a version of the workbook that teaches and supports signups without exposing private or messy internal project material.

## Public copy should keep

- Home / Start Here
- Signups
- Party assignment pages, if cleaned
- Build Library
- Coverage Checklist
- Role Quick Notes
- Healer Doctrine
- Great Fire Control Formation Guide
- Training Curriculum
- VOD Review Checklist
- Website Hub, if it points to public links

## Public copy should remove or hide

- private player names
- officer-only notes
- incomplete admin tabs
- raw doctrine backlog
- private VOD references
- internal feedback or critique
- tabs that look broken or experimental

## Recommended workflow

1. Duplicate the workbook.
2. Rename it as a public/review copy.
3. Remove private player data.
4. Hide or delete admin-only tabs.
5. Check formulas and dropdowns.
6. Test in Google Sheets.
7. Add website link once published.
8. Share with trusted reviewers first.

## Public release warning

Never publish the workbook just because the website is ready. The workbook needs its own privacy and usability pass.
