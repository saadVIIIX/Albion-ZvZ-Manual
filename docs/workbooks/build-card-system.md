# Build Card System

Build cards should make builds easier to read without turning the public site into an unverified meta list.

## Purpose

Build cards are visual references for roles that are already supported by a workbook package. They should not replace the workbook's live build library.

## What a build card should show

- role / weapon
- helmet
- chest
- boots
- cape
- food
- potion
- core ability notes
- role purpose
- timing notes
- common failures
- workbook package link
- status: draft / tested / stable / outdated

## What a build card should avoid

- pretending to be universal meta
- hardcoding patch-sensitive assumptions without review
- replacing officer judgment
- spreading private guild-specific critique publicly

## Visual asset plan

The user has item icons and Great Fire comp build screenshots. These should be used later when the visual asset workflow is stable.

For now, build-card pages can exist as placeholders and text templates.
