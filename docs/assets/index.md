# Assets and Visuals Plan

The website will eventually use item icons, build screenshots, and diagrams. For now, the asset plan is documented but not fully implemented.

## Suggested folders

```text
docs/assets/images/items/
docs/assets/images/builds/
docs/assets/images/diagrams/
docs/assets/images/workbook/
```

## Image use rules

- Use icons to clarify builds, not replace text.
- Avoid uploading copyrighted or private images unless they are safe to publish.
- Do not expose private guild names/player critique through screenshots.
- Keep build cards patch-sensitive and reviewable.

## Future diagram ideas

- crescent pressure shape
- kite route with turn window
- danger-area healing
- support defensive layering
- choke/door danger box
- first-through outside angle

## Current visual milestone

Website v1.2 adds the first original SVG draft diagrams and a custom stylesheet.

The current diagrams are teaching placeholders, not final art. They establish the visual language for future diagrams and build cards.

## Item icon policy

For item icons, use a consistent tier/color theme once visual build cards are implemented. The existing user-provided icons may be mixed-tier and should not be treated as the final public icon set.

See [Item Icon Sourcing and Tier Policy](item-icon-policy.md).

## Build-card asset readiness

Do not bulk-source icons yet. First choose the icon tier theme and confirm the build-card visual layout.

See [Build Card Asset Readiness](build-card-asset-readiness.md).
