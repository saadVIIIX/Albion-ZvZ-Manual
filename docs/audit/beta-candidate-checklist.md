# Beta Candidate Checklist

This checklist defines what must be true before releasing **Beta Candidate 1**.

## Website requirements

- Homepage is readable and does not show raw markup.
- Start Here links work.
- New-player path is clear.
- Role pages use broad public role language.
- Fight Style Literacy does not promise full comp packages.
- Weapon Notes are marked incomplete where needed.
- Applied Build Cards are clearly package-specific.
- Known Limitations are visible.
- Sources and Credit are balanced.
- No private names or unlisted VOD identifiers are exposed.

## Workbook requirements

- Master workbook is not shared publicly.
- Website Hub versions are synced.
- Public Copy Checklist exists.
- Feedback Triage exists.
- Build Card Mapping exists.
- Separation Tracker / Weapon Notes Tracker exists.
- Public workbook link is either pending or points to a cleaned copy.

## Review package requirements

- Review Pack is current.
- Release Readiness packet is current.
- Final Trusted Review Route is current.
- Public Sharing Note Template is current.
- Go/no-go page is current.

## Beta Candidate decision

A Beta Candidate is allowed to have rough edges. It should not have:

- preventable privacy problems
- unclear credit
- broken entry links
- workbook/control-comp overreach
- unmarked patch-sensitive build claims
