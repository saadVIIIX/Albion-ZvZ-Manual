# Page Quality Checklist

Use this checklist before treating any page as ready for public review.

## Structure

- Does the page have a clear audience?
- Does the page have a clear purpose?
- Does it start with a useful summary?
- Does it use headings instead of one giant wall of text?
- Does it link to related pages?
- Does it avoid mixing workbook-specific instructions into public doctrine?

## Doctrine quality

- Is the core claim supported by AO University/Khelron material, Albion mechanics, or explicit workbook application?
- If the page includes interpretation, is it labeled as project synthesis rather than source fact?
- Does the page explain behavior, not only definitions?

## Patch sensitivity

- Are mechanics values avoided unless verified?
- Are item/build claims marked as patch-sensitive?
- Is there a reminder to check Albion Wiki, patch notes, or in-game tooltip where needed?

## Privacy

- Are private VOD examples anonymized?
- Are player names removed unless intentionally public?
- Does the example teach the pattern instead of shaming the person?

## Readability

- Can a new player understand the first half?
- Can a role lead still get depth from the second half?
- Does the page avoid unnecessary jargon or explain it?
