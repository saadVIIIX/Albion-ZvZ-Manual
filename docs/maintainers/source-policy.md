# Source Policy

The project uses several source layers, but they do not all serve the same purpose.

## Source hierarchy

| Source layer | Best use |
|---|---|
| AO University / Khelron | Core doctrine backbone |
| Private/unlisted AOU guild VODs | Anonymous practical patterns and examples |
| Albion Wiki | Mechanics verification and patch-sensitive reference |
| Albion forums/community | Community context and gap analysis |
| EVE-style documentation sites | Information architecture reference |
| Workbook application | Specific operating standards for content |

## Public doctrine rule

The public manual should not pretend every workbook decision is universal Albion doctrine.

Example:

- Public site: "Holy and Nature both matter; healer mix is more than a count."
- Workbook: "For this comp/package, enforce this healer standard."

## Forum/community rule

Forum posts and random community discussions can reveal common questions, terminology, and gaps. They should not override stronger doctrine unless validated.

## Mechanics rule

Mechanics pages should link to Albion Wiki or patch-sensitive references and explain behavior. Exact values should be verified near publication.
