# Homepage and Navigation Standard

The homepage should not try to teach the whole manual. It should route readers.

## Homepage jobs

1. Explain what the project is.
2. Credit AO University/Khelron clearly.
3. Give reader paths.
4. Link to the major sections.
5. Explain the website/workbook/archive model.
6. Make current status obvious.

## What not to do

- do not turn the homepage into a changelog
- do not expose private archive content
- do not bury the new-player path
- do not make build cards look like universal meta before review

## Navigation principle

A reader should be able to reach one useful path within two clicks:

- New player → New Line Member path
- Role player → Role Guides / Learning Paths
- Officer → Training / Workbooks / Review
- Maintainer → Maintainer Guide
