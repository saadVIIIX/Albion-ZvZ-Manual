# Publishing Roadmap

The project should not wait until every page is perfect before testing the website structure, but it should also not publish a messy draft as final.

## Stage 1 — Internal draft

Current stage.

- source zip
- preview zip
- blueprint
- archive
- workbook package

## Stage 2 — Trusted review

Share preview with trusted friends or role leads.

Ask them:

- Can you find the page you need?
- Is the role guidance understandable?
- Is anything obviously wrong?
- Is the source/credit tone respectful?
- Is the workbook connection clear?

## Stage 3 — Public beta

Publish a clearly labeled beta site.

Requirements:

- no private names
- clear Khelron/AO University credit
- mechanics pages marked patch-sensitive
- workbook links reviewed
- obvious draft pages labeled

## Stage 4 — Stable release

After feedback, clean navigation, polish examples, add diagrams/build cards, and create a public update note.
