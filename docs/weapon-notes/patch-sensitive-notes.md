# Patch-Sensitive Notes

The manual should not be afraid to use weapon and item examples. Examples help players understand how tools work.

Patch sensitivity matters when the guide makes claims about exact values, current behavior, or current strength.

## High patch sensitivity

Treat these as requiring current verification:

- cooldown values
- exact ability numbers
- item ability behavior after patches
- current meta status or strength claims
- item tier/enchantment recommendations
- build substitutions
- weapon-specific interactions

## Lower patch sensitivity

These are usually more stable:

- do not clump
- damage must land on time
- healer range matters
- defensive layers should cover the danger area
- support utility should align with damage
- tanks should stop enemy paths before damage lands
- terrain changes fight value
- movement is the path to a useful position
- tools should create value for the zerg, not only personal safety or meter value

## Publishing rule

Before marking a weapon/build page stable, verify exact values and item behavior through current Albion references when those details matter.

Do not slow the whole public manual down for every illustrative example. A general example can stay useful if it is clearly teaching function rather than making a current-meta claim.

## Better wording

Use:

> This tool can illustrate area denial and path control.

Avoid:

> This is currently the best tool and every zerg should run it.

Use:

> Check current cooldown and item behavior before using this as a build recommendation.

Avoid:

> This exact cooldown/meta status is permanently true.
