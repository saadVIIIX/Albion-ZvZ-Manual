# Support Weapon Notes

Support weapons and utility tools make other players' value work.

## Broad support jobs

| Job | General value |
|---|---|
| Cleanse / debuff removal | keeps players functional through pressure |
| Resistance / shielding | helps the danger area survive |
| Purge | removes key enemy defensive/offensive value |
| Pierce / debuff | makes allied damage matter more |
| Anti-heal | weakens enemy recovery |
| Interruption | stops channels, engages, or key enemy actions |
| Area denial | changes enemy pathing and timing |

## How these tools create value

### Locus

**Role family:** defensive support / cleanse layer  
**General value:** Locus-style defensive support should answer real pressure on the real danger area.

Common mistakes:

- panic-dropping on safe space
- using it before pressure is real
- missing the group that is actually being bombed

### Oathkeepers

**Role family:** resistance/support layer  
**General value:** Resistance/shield support is strongest when it stays close enough to the threatened group and supports the people making the call possible.

Common mistakes:

- leaving tanks/frontline too early
- shielding safe backline while the danger area dies
- treating support as self-preservation only

### Rootbound

**Role family:** defensive support / stabilizer  
**General value:** Rootbound-style support is a teaching example for defensive stability, reset support, and keeping the zerg functional through pressure. Applied package priority belongs in the Formation Ledger.

### Truebolt

**Role family:** experimental support/utility  
**General value:** Truebolt remains an applied-package/experimental example. Use it to discuss how experimental utility should be labeled, not as general public doctrine.

### Purge roles: 1h Arcane / Heavy Mace / Lifecurse-style logic

**Role family:** purge/stop utility  
**General value:** Purge roles should identify enemy bomb weapons and remove important buffs before damage lands.

Common mistakes:

- purging after the enemy bomb already lands
- not preselecting important enemy weapons
- using purge on low-value targets while enemy damage setup is visible

## Timing rule

Support utility should land before or with the event it supports.

Late utility often looks active but fails to change the fight.
