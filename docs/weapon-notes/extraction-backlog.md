# Weapon Extraction Backlog

This backlog prevents the project from pretending we have complete weapon coverage before source-backed notes are extracted.

## Why this matters

The AO University channel includes more specific weapon, armor, and ability explanations than the current workbook/control-comp material.

Those notes should be recovered, but not guessed.

## Backlog categories

| Category | Current state | Action |
|---|---|---|
| Axes | under-extracted | recover AOU notes before writing detail |
| Swords | under-extracted | recover AOU notes before writing detail |
| Curses | under-extracted | recover AOU notes before writing detail |
| Crossbows | under-extracted | recover AOU notes before writing detail |
| Daggers / executes | partly covered through execute concepts | recover specific tool notes |
| Arcane tools | partly covered through purge/stop notes | expand from AOU source |
| Maces / hammer tools | partly covered through stopper/engage notes | expand from AOU source |
| Nature weapons | partly covered | expand specific weapon/ability notes |
| Holy weapons | partly covered | expand specific weapon/ability notes |
| Armor/helm/boot utility | partly covered | extract more item-specific notes |

## Entry format

When adding a new note, use:

```text
Tool:
Role family:
Source confidence:
General value:
AOU emphasis:
Common mistakes:
General or comp-specific:
Patch sensitivity:
Related pages:
```

## Do not do

- do not make up full comp packages
- do not imply a weapon is meta without current verification
- do not mix workbook-specific builds into general doctrine
- do not force all weapon notes into the current Great Fire Control Formation package
