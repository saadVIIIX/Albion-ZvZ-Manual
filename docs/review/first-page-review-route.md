# First-Page Review Route

This is the shortest useful review route for someone opening the project for the first time.

The goal is to test whether the site explains itself before asking reviewers to inspect deeper doctrine pages.

## Route A — New player first impression

Read in order:

1. [Home](../index.md)
2. [New to ZvZ](../start-here/new-to-zvz.md)
3. [Ten Rules Before CTA](../start-here/ten-rules-before-cta.md)
4. [How to Read Calls](../start-here/how-to-read-calls.md)
5. [New Line Member Learning Path](../learning-paths/new-line-member.md)

Review question:

> Does this tell a normal player what to do next?

## Route B — Role player first impression

Read in order:

1. [Role Guides](../roles/index.md)
2. Your relevant role page
3. One related fight concept
4. One related practical example

Review question:

> Does this teach the role as behavior, not only as a build?

## Route C — Officer first impression

Read in order:

1. [Training and VOD Review](../training/index.md)
2. [VOD Review Method](../training/vod-review-method.md)
3. [Workbooks and Tools](../workbooks/index.md)
4. [Workbook Public-Copy Strategy](../workbooks/public-copy-strategy.md)

Review question:

> Could an officer use this to run better content without reading the whole site?

## Route D — Trust and release check

Read in order:

1. [Sources and Credit](../sources-credit/index.md)
2. [Known Limitations](known-limitations.md)
3. [Public Beta Risk Check](public-beta-risk-check.md)
4. [Item Icon Policy](../assets/item-icon-policy.md)

Review question:

> Does the project feel honest, credited, and safe to share?
