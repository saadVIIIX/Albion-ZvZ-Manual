# Public Beta Risk Check

Before wider sharing, review risks in four categories.

## 1. Privacy risk

- private player names
- private guild names
- identifiable criticism
- screenshots with names
- unlisted VOD details that should not be public

## 2. Accuracy risk

- hardcoded mechanics values
- current item/build claims
- overgeneralizing one comp
- treating workbook choices as universal doctrine
- unsupported forum/community claims

## 3. Tone risk

- too much hero worship
- too little credit
- sounding like the project owns Khelron's doctrine
- sounding hostile to normal players
- making Nature/Holy arguments too combative instead of educational

## 4. Usability risk

- no clear start path
- too many pages without routing
- build cards look like meta claims
- workbook links expose admin tabs
- diagrams confuse instead of clarify

## Release rule

A public beta can still have rough edges, but it should not have preventable privacy or trust failures.
