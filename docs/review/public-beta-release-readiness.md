# Public Beta Release Readiness

This page decides whether the project is ready for a small public beta.

A public beta does not mean the site is finished. It means the project is safe enough, clear enough, and useful enough to share beyond the immediate project circle.

## Release standard

A public beta must be:

- honest about draft status
- clear about AO University / Khelron credit
- private-safe
- readable for new players
- useful for role leads
- careful with mechanics/build claims
- clear that workbook packages are optional tools, not universal doctrine

## Go / no-go checklist

| Area | Go condition |
|---|---|
| Credit | Khelron/AO University credited clearly without overdoing it |
| Privacy | no private player names, private guild critique, or unlisted VOD exposure |
| Mechanics | exact values marked for verification or avoided |
| Builds | build cards marked workbook-backed and patch-sensitive |
| Navigation | new-player and role paths are easy to find |
| Workbook | public-copy strategy exists and master workbook is not shared by accident |
| Review | feedback form, triage board, limitations, and review routes exist |
| Tone | educational, respectful, not guild-recruitment-coded |

## Current recommendation

The site is approaching trusted-review readiness. Before full public beta, it should receive at least:

- one healer review
- one tank/support review
- one DPS review
- one new-player readability review
- one mechanics/build sanity check

## Public beta warning

Do not publish the master workbook. Link only a cleaned public copy or mark workbook links as pending.
